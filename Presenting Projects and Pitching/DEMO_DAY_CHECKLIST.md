# Demo Day Checklist

Use this checklist to ensure your presentation is complete, polished, and ready for demo day.

---

## 1 Week Before

### Content
- [ ] Slide deck is complete (all slides drafted)
- [ ] Problem statement is clear and compelling
- [ ] Solution is explained at the right level for the audience
- [ ] Demo or screenshots are prepared
- [ ] Results/metrics are included
- [ ] Challenges and learnings are documented
- [ ] Contact info and links are on final slide

### Technical
- [ ] Demo environment is set up and working
- [ ] Demo has realistic test data (not "test123")
- [ ] All links in slides work
- [ ] Code is in a shareable state (GitHub repo, clean README)

### Practice
- [ ] Timed run-through completed at least once
- [ ] Identified sections that run long
- [ ] Know your transitions between slides

---

## 3 Days Before

### Content Refinement
- [ ] Slides reviewed for typos and errors
- [ ] Removed unnecessary slides or content
- [ ] Each slide has one clear message
- [ ] Font sizes are readable from the back of the room
- [ ] Visuals are high quality (not blurry or pixelated)

### Demo Preparation
- [ ] Demo script is written (exact steps you'll follow)
- [ ] Backup demo recorded (video or screenshots)
- [ ] Tested demo on the actual computer you'll use
- [ ] Browser bookmarks set up for quick navigation
- [ ] Sensitive info hidden (API keys, passwords, personal data)

### Practice
- [ ] Full presentation practiced at least 3 times
- [ ] Practiced with a timer — fits within time limit
- [ ] Practiced transitions and what to say between slides
- [ ] Received feedback from at least one person

---

## 1 Day Before

### Logistics
- [ ] Know the exact time and location
- [ ] Know how long you have to present
- [ ] Know the presentation order (when you go)
- [ ] Confirmed A/V setup (projector, microphone, adapters)
- [ ] Have required adapters (HDMI, USB-C, etc.)

### Backups Ready
- [ ] Slides saved locally on your laptop
- [ ] Slides uploaded to cloud (Google Drive, Dropbox)
- [ ] Slides on USB drive as backup
- [ ] Demo video recorded and accessible
- [ ] Screenshots ready if demo fails

### Final Review
- [ ] One more full run-through
- [ ] Timed and within limit
- [ ] Confident with Q&A responses
- [ ] Notes prepared (if using them)

---

## Day Of — Before You Present

### Setup (30+ minutes early)
- [ ] Arrive early to test setup
- [ ] Connect laptop to projector/display
- [ ] Test that slides display correctly
- [ ] Test audio if needed (for videos)
- [ ] Check internet connection if demo needs it
- [ ] Close unnecessary apps and notifications
- [ ] Set phone to silent/Do Not Disturb

### Final Prep
- [ ] Water nearby
- [ ] Notes accessible (if using)
- [ ] Clicker/remote tested (if using)
- [ ] Know how to advance slides
- [ ] Timer visible or someone keeping time

### Mental Prep
- [ ] Take a few deep breaths
- [ ] Remember: you know your project better than anyone
- [ ] Nervousness is normal — use it as energy
- [ ] Visualize a successful presentation

---

## During Your Presentation

### Opening
- [ ] Smile and make eye contact
- [ ] Speak clearly and not too fast
- [ ] Hook the audience in the first 30 seconds

### Throughout
- [ ] Make eye contact with different parts of the room
- [ ] Pause at key points (don't rush)
- [ ] Check the time periodically
- [ ] If something goes wrong, stay calm and move on

### Demo
- [ ] Narrate what you're doing
- [ ] Go slowly enough for audience to follow
- [ ] If it fails, switch to backup immediately
- [ ] Don't apologize excessively — just continue

### Closing
- [ ] End with a strong conclusion
- [ ] Thank the audience
- [ ] Invite questions confidently

---

## Q&A

- [ ] Listen to the full question before answering
- [ ] Repeat or paraphrase if needed
- [ ] Keep answers concise
- [ ] It's okay to say "I don't know" or "Great question, I'd need to look into that"
- [ ] If a question is off-topic, offer to discuss after

---

## After Your Presentation

- [ ] Thank anyone who helped
- [ ] Follow up with people who expressed interest
- [ ] Note feedback for future presentations
- [ ] Celebrate — you did it!

---

## Emergency Procedures

### Slides Won't Display
1. Try a different cable/adapter
2. Use cloud version on a different computer
3. Present from USB drive
4. Worst case: describe slides verbally, focus on demo

### Demo Crashes
1. Stay calm — don't apologize profusely
2. Switch to backup video immediately
3. If no video, use screenshots
4. Narrate what would happen: "Here's where it would show..."

### Internet Goes Down
1. Use pre-loaded/cached version if possible
2. Switch to screenshots or video
3. Explain the demo verbally

### You Forget What to Say
1. Pause and take a breath
2. Look at your slide — it should remind you
3. Glance at notes if you have them
4. It's okay to say "Let me take a moment..."
5. Skip to the next point if needed

### You Run Out of Time
1. Skip to conclusion slide
2. Summarize remaining points in 1-2 sentences
3. Don't rush through everything — better to end gracefully
4. Offer to discuss remaining content in Q&A
